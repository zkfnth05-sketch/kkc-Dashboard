# 내일 대회 관리 페이지 작업 계획

## 1. 대회 관리 페이지 워드프레스 연결 (숏코드)
현재 행사 관리(Event)를 보여주는 방식과 동일한 구조를 재사용합니다.

- **방법**: 워드프레스 숏코드(예: `[kkf_competition]`)를 새로 생성하거나, 기존 숏코드에 `view=public_competition` 파라미터를 추가하여 호출합니다.
- **결과**: React 앱이 파라미터를 읽어 '관리자 모드'가 아닌 **'사용자용 대회 리스트'**를 화면에 렌더링합니다.

## 2. "신청자 관리" 대신 "신청하기" 버튼으로 변경
사용자 권한에 따라 리스트의 버튼 기능을 분기 처리합니다.

- **디자인 로직**: React 코드에서 현재 접속자가 '일반 사용자'임을 감지하면 관리용 버튼(수정, 삭제, 신청자 명단)을 숨깁니다.
- **버튼 교체**: 관리용 버튼 대신 시인성이 좋은 **[신청하기]** 버튼을 배치합니다.

## 3. 사용자 직접 신청 기능 구현 (신청 폼)
[신청하기] 버튼 클릭 시 동작을 구현합니다.

- **신청 폼 팝업**: 해당 대회의 정보를 자동으로 불러와 사용자 정보(이름, 연락처, 견종 등)를 입력할 수 있는 **Modal(입력 창)**을 띄웁니다.
- **데이터 저장**: 입력된 데이터를 이미 구축된 대회별 신청자 테이블(`dogshow_applicant`, `stylist_applicant` 등)에 실시간으로 저장하는 API를 연결합니다.
- **관리자 연동**: 신청 완료 시 관리자 페이지의 "신청자 관리" 명단에 즉시 반영되어 통합 관리가 가능해집니다.

---
💡 **내일 작업 핵심**: "관리자는 신청자를 확인하고, 사용자는 신청을 하는" 양방향 시스템 완성!
오늘 카드 비율과 글자 크기 최적화를 완료했으므로, 내일은 이 안정적인 기반 위에 **'신청 시스템'** 기능을 탑재합니다.
(※ 대카테고리 명칭은 '독스포츠'로 통일하였습니다.)
